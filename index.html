Github에 블로그제작

  1. 깃허브에 계정만들기
  2. lein 설치하고 템플릿작성
  3. 블로그 프로젝트를 생성한다
  4. git clone한다
  5. config.edn 편집
  6. 서버 시작
  7. 마크다운으로 기사 작성하기
  8. github에 올리기

깃허브에 계정만들기

먼저 github.com에 가입하고 저장소를 만든다
저장소는 계정명.github.io라고 작성한다

https://help.github.com/articles/what-are-github-pages/

lein 설치하고 템플릿작성

leingen (레닝겐) 을 설치한다

블로그 프로젝트를 생성한다

cryogen을 이용해서 블로그 프로젝트를 생성한다

lein new cryogen my-blog

git clone한다

public 디렉토리는 없기때문에 작성한다

예) git@github.com:[계정명]/[생성한 저장소] [target 디렉토리]  
git clone git@github.com:clonekim/clonekim.github.io.git resources/public

config.edn 편집

resources/templates/config.edn를 편집한다
내용중 아래와 같이 ""으로 설정한다

:blog-prefix ""

서버 시작

아래 명령어로 서버를 시작한다
서버는 로컬에서 생성하고 앞으로 작성한 문서를 서버에서 파싱해주는 역할을 한다

lein ring server
서버를 시작하고 http://localhost:3000 에 접속한다

마크다운으로 기사 작성하기

resources/templates/md/posts에 yyyy-mm-dd-docs.md 형식으로 파일을 작성한다

{:title "TEST 기사"
 :layout :post
 :tags  ["태그1" "태그2"]
 :toc true}
여기에 마크다운문법을 이용해서 작성한다...
## Hello

파일을 편집하고 저장 할 때 마다 서버에서 자동으로 마크다운 페이지를 파싱하여
resources/publc 밑에 결과물을 생성해 낸다 결국 public 디렉토리만 github에 올리면 끝

github에 올리기

git add --all
git commit -m "cryogen 적용"
git push -u origin master
